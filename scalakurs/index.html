scala

et moderne, statisk typet, objekt-funksjonelt programmeringsspråk på jvm-en

  • mer moderne språk enn java
  • hybrid objektorientert og funksjonelt språk og påstår å forene det beste fra to verdener

class person(val name: string, var age: int)

  • klasser deklareres med class
  • instansieres med new
  • paranteser er valgfritt hvis det ikke er constructor-parametre
  • public er default
  • val betyr medlemsvariabel/instansvariabel med getter, men ingen setter fordi den ikke kan reassignes (final i java)
  • var betyr det samme, men med setter også
  • accessors har samme syntaks som felter (ikke get/setxxx)

constructors

primary constructor


class person(age: int) {
  val category = if (age < 35) "young" else "old"

  def sayhi = "hi, i'm " + category
}

new person(30).sayhi //hi, i'm young

auxillery constructors


class date(val time: long) {

  def this(date: date) = this(date.time)
}
  • age: int er default constructor
  • body til constructor er hele klasse-bodyen
  • category er en medlemsvariabel med public getter
  • age er ikke en medlemsvariabel siden den ikke har val/var og ikke brukes annet enn i constructor
  • ekstra constructors med def this()
  • ekstra constructorer kan man ha så mange man vil, men må kalle en annen eller default constructor

companion-objekter


class person(val name: string, val age: int)

//i samme fil
object person {
  def apply(name: string, age: int) = new person(name, age)
}

val p = person("frode", 33)
  • objekter - singleton objekter - en klasse med bare 1 instans
  • kan sammenlignes med static i java, men er ikke helt det
  • kalles med klassenavnet
  • companion objekter er et objekt med samme navn som en klasse i samme fil
  • her legger man alt som tilhører klassen og ikke instansen (igjen veldig likt static i java)
  • vanlig å legge factory-metoder sånn som her
  • apply metoden har spesiell betydning i scala og bli kalt hvis vi kaller på navnet på objektet

list

scala.immutable

  • implementerer isempty, head og tail.
  • o(n) på de fleste operasjoner
  • o(1) på prepend og head/tail-akess
  • egnet for rekursive løsninger basert på head:tail-akess
  • eksempler: 
    scala> 4 :: list(1, 2, 3) // prepend, o(1)
res0: list[int] = list(4, 1, 2, 3)
    scala> list(1, 2, 3) ::: list(4) // append, o(n), foretrekk prepend
res1: list[int] = list(1, 2, 3, 4)

map()

  • map() tar inn en funksjon og utfører den på alle elementer
  • map() transformerer listen ihht til funksjonen gitt inn
scala> val numbers = list(1,2,3)
numbers: list[int] = list(1, 2, 3)

scala> numbers.map((x: int) => x + 1)
res0: list[int] = list(2, 3, 4)

filter()

  • filter() tar inn en funksjon (predikat)
  • filter() returnerer et subsett av listen hvor predikatet er sant
scala> val mylist = list(1,2,3,4,5,6)
mylist: list[int] = list(1, 2, 3, 4, 5, 6)

scala> mylist.filter((x: int) => x > 3)
res0: list[int] = list(4, 5, 6)

verktøy for funksjonell programmering

med pattern matching: 

def fact(n: int) = n match {
  case 0 => 1
  case n => n * fact(n - 1)

def fact(n: int) =
  if (n == 0) 1

case classes

(nesten) klasser med god støtte for pattern matching

eksporterer konstruktørparametere


case class person(firstname: string, lastname: string)
                    

scala> val person = person("ola", "nordmann")
person: person = person(ola,nordmann)

scala> person.firstname + " " + person.lastname
res0: string = ola nordmann
  • konstruktørparametere automatisk tilgjengelig etter instansiering
  • reduserer boilerplate-kode

pattern matching direkte på person


def printperson(person: any) {
  person match {
    case person(first, last) => println(first + " " + last)
    case _ => println("not a person!")
  }
}
                

scala> printperson("ola nordmann")
not a person!

scala> printperson(person("ola", "nordmann"))
ola nordmann
scoper konstruktørparameterene i pattern match blokken

pattern matching på lister


def checkiffirstpersonisola(persons: list[person]) =
  persons match {
    case list(person("ola", _), _*) => true
    case _ => false
  }
                

scala> val persons = list(person("ola", "test"),
    | person("test", "test"))
persons: list[person] = list(person(ola,test), person(test,test))

scala> checkiffirstpersonisola(persons)
res0: boolean = true
_* betyr villlkårlig mange følgende elementer
